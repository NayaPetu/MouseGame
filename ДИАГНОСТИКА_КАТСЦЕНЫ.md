# Диагностика: Почему кат-сцена сразу переходит к main

## Проблема
Кат-сцена IntroCutscene загружается, но через секунду автоматически переходит к сцене main.

## Причина
Это означает, что в методе `ImageCutscene.Start()` срабатывает одна из проверок, которая вызывает `Invoke(nameof(LoadNextScene), 1f)`.

**Возможные причины:**
1. ❌ **Image Display не назначен** в компоненте ImageCutscene
2. ❌ **Список Images пустой** (нет изображений для кат-сцены)

## Что нужно проверить в Unity Editor:

### 1. Откройте сцену IntroCutscene.unity

### 2. Найдите GameObject с компонентом ImageCutscene
- Обычно это GameObject с именем "ImageCutscene" или находится в Canvas

### 3. Проверьте компонент ImageCutscene в Inspector:

#### А) Image Display:
- ✅ Поле **Image Display** должно быть назначено
- Должна быть ссылка на UI Image компонент (обычно это Image или Image (Legacy))
- Если поле пустое - перетащите Image GameObject в это поле

#### Б) Images (список изображений):
- ✅ Список **Images** должен содержать хотя бы одно изображение
- Нажмите "+" чтобы добавить элемент в список
- Для каждого элемента:
  - **Image**: перетащите Sprite изображения
  - **Display Time**: время показа в секундах (например, 5)

#### В) Next Scene Name:
- ✅ Должно быть "main" (или имя следующей сцены)

### 4. После исправления:
1. Сохраните сцену (Ctrl+S)
2. Запустите игру
3. Нажмите "Начать"
4. В консоли должны появиться сообщения:
   - `[ImageCutscene] Start вызван!`
   - `[ImageCutscene] Image Display назначен: ...`
   - `[ImageCutscene] Найдено изображений: X` (где X > 0)
   - `[ImageCutscene] Запускаю корутину ShowCutscene()...`

**Если видите сообщение:**
- `[ImageCutscene] ОШИБКА: Image Display не назначен!` → назначьте Image Display
- `[ImageCutscene] ОШИБКА: Нет изображений для кат-сцены!` → добавьте изображения в список Images
