# БЫСТРОЕ ИСПРАВЛЕНИЕ: Анимации Letter и Recipe

## Проблема
Анимации не работают из-за **неправильного порядка компонентов**.

## Решение (2 минуты)

### Для префаба Letter:

1. Откройте префаб `Letter.prefab`
2. В инспекторе найдите компонент **Animator**
3. **Переместите Animator в конец списка компонентов:**
   - Перетащите компонент Animator мышкой вниз, чтобы он был **после** Rigidbody2D и ItemPickup
   - Или: удалите Animator (правой кнопкой → Remove Component) и добавьте заново в конце

4. **Проверьте настройки Animator:**
   - Controller: назначен ✓
   - Update Mode: Normal
   - Animate Physics: **выключена** (галочка не стоит)

5. Сохраните префаб (Apply)

### Для префаба Recipe:

1. Откройте префаб `Recipe.prefab`
2. Убедитесь, что компонент **Animator** находится **в конце** списка компонентов (после Rigidbody2D)
3. Проверьте настройки Animator (как выше)
4. Сохраните префаб (Apply)

## Правильный порядок компонентов:

1. Transform (автоматически)
2. Script компоненты (FriendNote/Recipe)
3. SpriteRenderer
4. Collider2D
5. Rigidbody2D ← физика перед анимацией
6. ItemPickup Script
7. **Animator** ← в самом конце!

## Почему это важно?

В Unity порядок компонентов может влиять на порядок выполнения. Animator должен обновляться после физики, поэтому он должен быть после Rigidbody2D.
